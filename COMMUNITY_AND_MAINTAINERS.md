# Community and maintainers

This doc defines the **contributor/maintainer path** and the boundary between **core** (main Ona repo) and **community** (this repo).

---

## Core vs community

| Scope | Where | What |
|-------|--------|-----|
| **Core** | Main Ona repo | App code, boot, systems, production scripts, core docs. Single source of truth for the app. |
| **Community** | This repo (onapublic) | Community policies, maintainer/contributor onboarding, code of conduct, security reporting. |

---

## Contributor path

- **Code and core docs:** Contribute to the **main Ona repo** (fork, branch, PR). Follow its CONTRIBUTING and style guide.
- **Skills:** Propose or add skills (SKILL.md / skill packages) as described in this repo (e.g. MAINTAINERS.md or a dedicated “How to add a skill” doc when added).
- **Install and setup improvements:** Propose changes via the main repo (scripts, install docs). For ideas or discussions, use this repo’s issues or docs when available.
- **Release and triage:** Documented in [MAINTAINERS.md](MAINTAINERS.md) when we adopt it.

---

## Maintainer path

- **Maintainer/contributor onboarding** (how to release, triage, add skills or docs) is in [MAINTAINERS.md](MAINTAINERS.md).
- The main Ona repo links to this repo so contributors and maintainers see the path.

---

## Community policies (Discord / other channels)

If Ona adopts a public community channel, **public policies** (code of conduct, security reporting, expectations) live in **this repo** ([CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) and related docs).
